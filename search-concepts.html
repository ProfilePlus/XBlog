Search Prototype Set

三套搜索原型已经放好，可以直接逐个打开比较。

它们都围绕“公开文章搜索 + AI 总结 + 引用来源”展开，但重点分别偏向策展感、可信度和阅读感。

Concept A

AI Summary First

最像公开站首页的延伸，把 AI 总结放在视觉中心，再承接结果列表和命中片段。

  • 优点：亲和、直观、最容易让用户理解“搜索 + 总结”。
  • 代价：排序逻辑没有 B 那么明显。
打开 A
Concept B

Evidence First

更像可信检索控制台，强调得分、信号、引用和回答边界，适合二期 RAG 演进。

  • 优点：最利于解释系统为何这样答、为何这样排。
  • 代价：气质会比站内其他页面更工具化。
打开 B
Concept C

Reading Lens

把搜索做成“找文章并进入阅读”的页面，更贴近 XBlog 现在的内容产品感。

  • 优点：最自然地鼓励点击文章继续深读。
  • 代价：证据结构和排序依据不如 B 显性。
打开 C
